# Enrichment ガイド — 各サイトを写真でリッチに

クライアント候補からの **「シンプルすぎる／空白が目立つ」** という印象を解消するための画像追加プラン。

## 監査結果

`public/images/` 配下に Codex が生成済み画像が大量にありますが、**コンポーネントから参照されていない**ものが多数あります。それを差し込むだけで、**コード修正のみで** 大幅に画面密度が上がります。それでも足りない部分は新規生成。

|---|---:|---|
| **chiro**（整体） | 6枚 + SVG 5枚 | ヒーロー画像、症状アイコン、施術写真をフル投入 |

## 二段階アプローチ

### Phase A — 既存画像をコンポーネントで使う（コード修正のみ）

各業種MDの「Phase A: 既存画像活用」セクションに、**どのファイルをどのセクションのどう差し込むか**を具体的に記載。Codex 不要。

### Phase B — 追加画像を生成（Codex指示書）

各業種MDの「Phase B: 追加生成」セクションに、新規生成すべき画像の**詳細プロンプト**を記載。`docs/plans/regenerate/` と同じ書式（プロンプトEN、サイズ、配置先、構図、避ける要素）で書く。

## 命名規則・運用フロー

`docs/plans/regenerate/README.md` と同じ：

- 生成形式: **WebP** quality 85（写真）／**SVG**（アイコン）
- 既存と同名で `public/images/<site>/` に上書き
- ファイル名・パスは MD 内で明示
- 生成サービス: Flux 1.1 Pro / Midjourney v6.1 / DALL·E 3
- 共通スタイル指針（フィルム感、自然光、人物は顔出ししすぎない）は regenerate ガイド準拠

## Phase A の実装は別タスクとして進める

Phase A（既存画像を使う）は **コードの単純な編集**なので、本ドキュメントを参照しながら順次コンポーネントを更新します（必要なら別途指示してください）。

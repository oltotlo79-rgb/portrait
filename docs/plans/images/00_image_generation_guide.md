# 00 画像生成 共通ガイド（Codex向け）

このフォルダ内の各MDファイルは、**Codex（画像生成AI）にホームページ用の写真・グラフィックを生成させるための指示書**である。各サイトに対応するMDを1つ読み込めば、そのサイトに必要な画像が全て揃うように作られている。

## 出力ルール

| 項目 | 内容 |
|---|---|
| 配置先 | `public/images/<site>/` （例: `public/images/restaurant/`） |
| 形式 | 写真系 → **WebP**（fallback JPEG）／グラフィック → **SVG**（不可なら PNG透過） |
| ピクセル密度 | 2x相当で書き出し（`-2x.webp`サフィックス） |
| メタデータ | EXIFは削除。`alt`テキスト案は各画像の項目に記載済 |
| 命名 | 各MDで指定されたファイル名を厳守 |

## 画質・サイズの基準

| 用途 | 基準サイズ（横幅） | アスペクト |
|---|---|---|
| ヒーロー（フル幅） | 2880px | 16:9 または 21:9 |
| ヒーロー（縦長スマホ重視） | 1440px | 4:5 |
| セクションのキービジュアル | 1920px | 3:2 |
| カード／グリッド | 1200px | 4:5 または 1:1 |
| サムネ（実績一覧用） | 1000px | 4:5 |
| OGP | 1200×630 | 固定 |
| favicon | 512×512 (SVG優先) | 1:1 |

## 共通スタイル指針

各サイトのトーンを守りつつ、サイト群全体として以下を共通とする：

- **広告くさい商業写真NG** — 笑顔の全員集合写真、CGっぽい合成、過度なライティング、ストックフォト感
- **被写体の顔は基本的に出しすぎない** — 横顔、後ろ姿、手元、シルエットを優先（ポートレート指定の場合のみ正面）
- **ナチュラルライト優先** — スタジオライトはフィットネスや高級飲食など指定時のみ
- **テキストを画像内に焼き込まない** — タイポはNext.js側で乗せる
- **過度な彩度・コントラストNG** — 上品なフィルムトーン寄り

## 顔写真（人物ポートレート）について

- 完全に**実在しない人物**として生成する
- 同一サイト内で複数人のポートレートを生成する場合は、雰囲気を揃える（光の方向、背景、衣装トーン）
- 商業写真感ではなく、**雑誌の取材ページ**のような自然さを目指す

## 避けるべきもの（全サイト共通）

- 西洋風の典型ストック家族写真
- 「ガッツポーズ」「OKサイン」「指差し」などのオーバーアクション
- 不自然な手の指（多指・歪み）
- 透かしやロゴが入った写真
- 文字が崩れた看板や本

## カラー指定の使い方

各サイト指示書のカラーパレットは、**画像内に同系色が支配的に映る**ように使う（例: 民泊なら深緑＋生成り、フィットネスなら黒＋ネオンイエロー）。

## ファイル命名規則

```
{番号}-{用途}-{バリエーション}.{ext}

例:
01-hero-day.webp
01-hero-night.webp
02-room-ume.webp
03-detail-bowl.webp
```

各MDの「必要画像リスト」の表に**ファイル名カラム**を必ず明記してある。Codexにはこの名前で書き出させること。

## 共通画像（`public/images/common/`）

`90_portfolio_images.md` で扱う、サイト群全体で共有する画像群：

- ファビコン
- OGPベース
- ノイズテクスチャ
- カーソル用SVG

## 各サイト指示書の構成

各 `1X_*_images.md` は以下の構成で書かれている：

1. サイトのトーンの再確認（短く）
2. ライティング・色温度
3. 必要画像リスト（表 + 各画像のプロンプト指示）
4. やってはいけないこと
5. 共通装飾（SVG等）

これに従って画像を量産すること。
